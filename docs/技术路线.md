# 基于 RAG 与 MCP 的智能 IT 运维 Agent 技术路线

## 1. 总体架构

用户问题进入 Agent 后，先由 Prompt Router 判断意图，再进入 RAG 检索或 MCP 工具规划。知识类问题直接基于检索片段生成回答；操作类问题先结合 SOP 生成执行计划，再由 MCP 工具层调用工单系统、账号系统或系统控制器。高风险操作经过 Human-in-the-loop 审批后才执行。

## 2. 模块划分

- 数据层：`data/knowledge/` 存放 Markdown 运维知识手册。
- 索引层：`it_ops_agent/knowledge_base.py` 负责加载、切分、保存索引。
- 检索层：`it_ops_agent/retrievers.py` 负责 BM25、向量检索和混合召回。
- 重排层：`LightweightReranker` 负责对候选片段重排。
- 路由层：`it_ops_agent/router.py` 负责识别知识问答、工单查询、密码重置、服务重启等意图。
- 工具层：`it_ops_agent/mcp_tools.py` 以 MCP 风格封装外部系统能力。
- 审批层：`it_ops_agent/hitl.py` 控制高风险操作是否允许执行。
- 编排层：`it_ops_agent/agent.py` 负责组织完整流程并输出答案。

## 3. 当前原型与生产替换

当前原型不依赖外部服务，方便本地运行和验收。生产环境建议替换如下：

- 本地 Markdown 索引替换为 Qdrant Collection。
- 哈希向量替换为企业 embedding 模型。
- 轻量 Rerank 替换为 BGE Reranker 或私有 CrossEncoder。
- 内存 MCP 工具替换为真实 MCP Server，对接 ServiceNow/Jira、LDAP/IAM、Kubernetes、Ansible 等。
- 本地审批策略替换为企业 IM 或工单审批流。

## 4. 安全策略

查询类动作默认自动执行；追加备注、密码重置、服务重启、配置变更等动作必须经过人工确认。工具调用结果统一记录工具名、参数、风险级别和执行结果，便于审计。
